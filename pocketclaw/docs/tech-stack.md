# Tech Stack

## Primary direction

PocketClaw is currently planned as a Flutter-based mobile client.

## Why Flutter

- one codebase for Android and iOS
- strong support for adaptive layouts
- good fit for chat-heavy UI
- straightforward path toward shared phone and future wearable-facing logic
- clean integration point for reusable Dart packages

## Current note

The repository can be structured for Flutter before the Flutter toolchain is available on every development host.

That is the current state of this repository: the monorepo scaffold can move forward now, while full generated project validation will follow once Flutter/Dart are available locally or in CI.
